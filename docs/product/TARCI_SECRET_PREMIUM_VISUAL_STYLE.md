# Tarci Secret Premium Visual Style

## 1. Definición visual oficial

Dirección visual oficial de Tarci Secret:

- **Premium Warm Minimalism**
- **Minimalismo cálido premium**

Tarci Secret debe verse:

- clara;
- intuitiva;
- elegante;
- cálida;
- moderna;
- familiar;
- privada;
- minimalista;
- premium;
- con alma propia de Tarci Secret.

Tarci Secret no debe verse:

- lila pastel como identidad principal;
- navideña como estética permanente;
- infantil;
- genérica;
- corporativa fría;
- saturada;
- como panel técnico;
- como formulario administrativo.

La experiencia visual debe transmitir:

- secreto;
- regalo;
- familia;
- confianza;
- privacidad;
- ilusión;
- grupo real;
- facilidad.

Sin limitar la marca a Navidad ni a campañas estacionales.

## 2. Paleta recomendada

Guía base de color:

- Fondo principal: Warm Ivory `#FAF7F1`
- Superficie/card: Soft Cream `#FFFDF8`
- Texto principal: Warm Charcoal `#2E2A27`
- Primario (marca): Deep Plum / Secret Aubergine `#3A223F` o `#42283E`
- Acento emocional: Soft Terracotta `#C96F53`
- Acento premium: Muted Gold `#C7A76C`
- Éxito/listo: Sage Green `#7A9B7A`
- Error/alerta suave: Soft Coral `#D85C4A`

Reglas de uso de color:

- El lila/lavanda no debe ser color dominante.
- El morado profundo sí puede representar marca, secreto y jerarquía principal.
- El dorado debe usarse poco, solo como acento premium puntual.
- El terracota debe funcionar como acento cálido y humano.
- Evitar azul genérico como color principal de identidad.

## 3. Uso del logo/icono

Uso recomendado:

- splash;
- dashboard;
- estados vacíos;
- momentos especiales (confirmaciones o hitos de flujo).

Reglas:

- no repetir logo/icono en todas las cards;
- no deformar proporciones;
- no usarlo como decoración sin propósito funcional;
- si el logo actual pierde legibilidad en tamaños pequeños, preparar una variante reducida en una fase posterior (sin cambiar assets en este ciclo documental).

## 4. Estilo de pantallas

Cada pantalla debe cumplir:

- una decisión principal;
- una card protagonista;
- mucho aire visual;
- títulos humanos;
- CTA claro;
- textos cortos;
- cero términos técnicos visibles para usuario final.

Patrón de lectura recomendado:

1. dónde estoy;
2. qué falta;
3. qué puedo hacer ahora;
4. qué pasa al tocar la acción principal.

## 5. Componentes visuales esperados

### 5.1 Pantalla de bienvenida

- Hero limpio, cálido, no infantil.
- CTA principal único: iniciar flujo core.
- CTA secundaria discreta: unirse por código.

### 5.2 Wizard

- Una pantalla = una decisión.
- Progreso visible y simple.
- Botones grandes, texto corto, retroalimentación clara.

### 5.3 Detalle del grupo

- Card protagonista de estado del grupo.
- Resumen de preparación en formato claro y escaneable.
- Acción principal contextual: completar pendiente o sortear.

### 5.4 Checklist de preparación

- Estados explícitos: pendiente, en preparación, listo para sortear.
- Señales de color moderadas (sin saturación decorativa).
- Bloqueos explicados con lenguaje humano.

### 5.5 Invitación por código

- Código visible y compartible sin fricción.
- Jerarquía clara entre ver, copiar y compartir.
- Mensajes de éxito/error entendibles.

### 5.6 Resultado privado

- Foco absoluto en privacidad.
- Presentación emocional sobria (premium cálida, no infantil).
- Evitar ruido visual alrededor del dato principal.

### 5.7 Secretos gestionados

- Diferenciar claramente “mis datos” de “datos que gestiono”.
- Limitar visibilidad al alcance funcional permitido.
- Legibilidad por encima de decoración.

### 5.8 Cards

- No todas iguales: jerarquizar por importancia.
- Superficies limpias y contraste cómodo.
- Evitar densidad de texto.

### 5.9 Botones

- Un botón primario dominante por pantalla.
- Secundarios discretos y consistentes.
- Tamaño cómodo para uso móvil familiar.

### 5.10 Chips

- Útiles para estado/categoría breve.
- No reemplazan títulos ni mensajes clave.
- Colores suaves, semántica consistente.

### 5.11 Modales

- Solo para decisiones relevantes o confirmaciones.
- Título humano + consecuencia clara.
- Máximo foco en una acción principal.

### 5.12 Bottom sheets

- Para acciones contextuales rápidas.
- Contenido breve y accionable.
- Evitar convertirlos en mini-menús complejos.

## 6. Qué copiar de apps competidoras

Sí copiar:

- claridad de flujo;
- una pantalla = una decisión;
- progreso visual;
- botones grandes;
- invitación clara;
- lenguaje humano.

No copiar:

- estética navideña permanente;
- azul dominante como identidad;
- wishlist en MVP;
- buscador de regalos en MVP;
- menú grande en MVP;
- notificaciones en MVP.

## 7. Reglas visuales obligatorias

1. No usar lila/lavanda como identidad dominante.
2. No usar Navidad como estética permanente.
3. No saturar con ilustraciones.
4. No mostrar textos técnicos.
5. No usar más de una acción principal por pantalla.
6. No repetir información innecesaria.
7. No usar cards iguales para todo.
8. No mostrar “Advanced settings” como bloque visible principal.
9. No mezclar idiomas.
10. No sacrificar claridad por decoración.

## 8. Criterio de aceptación visual

Una pantalla solo se considera aceptable si:

- se entiende en menos de 3 segundos;
- se ve limpia;
- se ve premium;
- usa lenguaje humano;
- tiene una acción principal clara;
- parece Tarci Secret, no una app genérica;
- no parece navideña salvo que el usuario elija una celebración navideña.
